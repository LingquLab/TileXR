# Remove MC2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove TileXR-owned MC2 code and make `ops-transformer` available only as an on-demand reference checkout.

**Architecture:** Delete the MC2 implementation and its dedicated build surface instead of leaving compatibility shims. Remove the tracked `ops-transformer` submodule, then register it with the existing reference downloader so `reference/ops-transformer` remains available without becoming a runtime or build dependency.

**Tech Stack:** Git submodules, Bash, Markdown, CMake source tests

## Global Constraints

- Delete the complete `examples/mc2` tree.
- Keep `ops-transformer` only as ignored, on-demand source at `reference/ops-transformer`.
- EP and collectives must not initialize, copy, include, or link `ops-transformer`.
- Keep historical documents under `docs/superpowers/` unchanged.
- Use `mutagen` for any server file transfer; this plan performs local validation only.
